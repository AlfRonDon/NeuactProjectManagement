Color System v1

One color, one meaning. Status colors signal severity; chart colors signal data. Never mix the two. All other tokens derive from this base.

01 — Foundation

Greys, surfaces, text

These do most of the work. Page chrome, body text, borders, dividers. If a UI element doesn't communicate status or data, it uses these tokens.

02 — Status

Five severity tiers

Used wherever the UI communicates "how is this going?" — project pills, alert badges, blocker callouts, task urgency. Five tiers, not four: warn (amber) and hot (orange) are different so "needs attention" doesn't read as urgently as "in trouble."

The Rule

Every place in the UI that communicates status uses one of these five tiers. "Behind by 4 days" → warn. "Overloaded" → bad. "Critical priority" → hot. No invented intermediate colors, no project-specific reds.

04 — Chart Series

Reserved for data, never for status

Use these for line charts, multi-series comparisons, anywhere the eye needs to track distinct quantities. Status colors overlay the chart for events (red dot = blocker, amber band = at-risk window) but the underlying series stay neutral. Planned/forecast/baseline lines are always dashed grey, never colored.

06 — Avatars

Hashed to chart series, never status

A user's avatar color is deterministic — hash their name into the five chart-series buckets. This keeps avatars visually distinct from status pills, so a person's identity color never gets mistaken for "this person is in trouble."

08 — Cheatsheet

Decision tree

When you reach for a color, walk this list top-down.

01   Is it text, a border, or a surface? → foundation
02   Does it communicate "how is this going?" → status (ok/info/warn/hot/bad)
03   Is it a tinted background showing a load level? → status @ 0.20–0.25 opacity
04   Is it a line, bar, or area in a chart? → chart series
05   Is it a task bar identifying a team? → chart series, keyed to team
06   Is it an avatar? → chart series @ -bg / -fg pair, hashed from name
07   None of the above? → you're inventing a color. Stop.
